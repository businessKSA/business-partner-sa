# منصة التوظيف — الرؤية وخطة التنفيذ

> المرجع التنفيذي لرؤية المنتج. آخر تحديث: 2026-07-23.

## الرؤية

أسهل نظام توظيف لصاحب العمل: **طلب توظيف واحد ← أفضل المرشحين ← مراجعة سريعة ← مقابلة ← عرض ← تعيين** — بدون تنقّل بين نماذج كثيرة.

ثلاثة أوضاع تشغيل لكل طلب:
1. **SELF_SERVICE** — صاحب عمل يدير التوظيف بنفسه.
2. **MANAGED_BY_BUSINESS_PARTNER** — فريق BP يتولى التوظيف نيابة عن العميل (Recruiter معيّن + SLA + قوائم مختصرة + إخفاء بيانات التواصل حتى الاعتماد).
3. **مؤتمت** — النظام يبحث ويطابق ويقترح، والإنسان يعتمد.

## القرار المعماري

- **Notion ليس قاعدة البيانات التشغيلية.** قاعدة الموقع (Production DB حقيقية) هي Source of Truth؛ Notion مصدر بيانات ومرجع إداري يُستورد ويُزامَن عبر Integration Layer منفصلة (انظر `hr-notion-mapping.md`).
- كل سجل مستورد يحمل `notionPageId/notionDatabaseId/syncStatus/lastSyncedAt/syncError`، مع منع التكرار في المزامنة.
- الواجهة لا تقرأ من Notion مباشرة ولا تصلها مفاتيحه؛ اليوم تتعامل حصراً مع طبقة `HRStore` (mock الآن، API لاحقاً بنفس التواقيع).
- **n8n للتكاملات فقط** (مزامنة Notion، بريد، واتساب، تذكيرات، تقويم، Webhooks، Retry/DLQ) — منطق الحالات والصلاحيات والمطابقة داخل التطبيق. كل Workflow يوثَّق (Owner/Trigger/IO/Error branch/Retry/Idempotency) قبل النشر.

## مصفوفة الموافقات البشرية (AI يقترح ولا يقرر)

يمنع التنفيذ التلقائي بدون اعتماد بشري في: نشر الوظيفة أول مرة، الرفض النهائي بسبب تقييم AI، رسائل الرفض، كشف بيانات مرشح لصاحب عمل، الراتب النهائي، إرسال/تعديل/سحب العرض، دمج سجلات مكررة، حذف بيانات مرشح.

## الأمان والخصوصية

عزل كامل بين العملاء (organizationId/companyId على كل صف)، RBAC، تسجيل العرض/التنزيل/المشاركة، تشفير نقل وتخزين، Signed URLs للمستندات، سياسة احتفاظ، حقوق تصحيح/حذف/تصدير للمرشح، سجل موافقات، Audit Log غير قابل للتعديل، لا خصائص حساسة في المطابقة إلا لضرورة قانونية، امتثال نظام حماية البيانات الشخصية السعودي (PDPL).

## مبدأ الواجهة

كل شاشة تجيب: ماذا يحدث الآن؟ ما الذي يحتاج تدخلي؟ ما الإجراء الأفضل التالي؟ («راجع 8 مطابقات قوية»، «اعتمد الوصف الوظيفي»...). لا مصطلحات تقنية للمستخدم (لا Webhook/Model/Queue) — «تعذر إرسال دعوة واحدة وسنعيد المحاولة تلقائياً».

## قواعد التنفيذ الملزمة

بدون بيانات وهمية إذا توفرت حقيقية (الواجهة الحالية موسومة «بيانات تجريبية» حتى الربط) · لا استيراد كامل لـ Notion دفعة واحدة · Dry Run وPreview دائماً · لا تعديل على Notion قبل تحديد اتجاه المزامنة · لا نشر Workflow قبل اختباره · الواجهة لا ترتبط بـ n8n مباشرة · AI لا يرفض نهائياً · لا خلط بيانات عملاء · لا إرسال جماعي أثناء التطوير · Feature Flags للميزات غير المكتملة · لا نشر Production قبل Preview واختبارات.
