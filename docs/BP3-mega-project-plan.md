## 🎯 نبذة تنفيذية

نعطي أي **رائد أعمال** فريق عمل جاهز من **اليوم الأول** — بدل ما ينتظر ويوظّف ويدفع رواتب. الفريق عبارة عن **موظفين أبطال بالذكاء الاصطناعي**، كل واحد متخصص (مبيعات، تسويق، قانوني، امتثال، تقنية، مشتريات…)، يشتغلون كطاقمه الفعلي، يتكاملون مع أنظمته (Notion / واتساب / إيميل)، وينفّذون بمبادرة.

نبيعها **باشتراك شهري (SaaS)**، تبدأ من السعودية. الهدف القريب: **بروتوتايب → تجربة مع ~50 عميل → إصلاح الأخطاء → تقديم على حاضنة/مسرّعة أعمال.**

## 🔌 طبقة التكاملات والأتمتة (قيمة أساسية للعميل)

نفس تجربة «كلود مربوط بكل أدواتك» نعطيها لكل عميل عبر وكلائه — Notion، إيميل، واتساب، تقويم، تخزين، تصميم، أتمتة — عشان يصير **أسرع وأقوى ومخرجاته مباشرة**. بدون التكاملات الوكيل يظل يتكلم فقط؛ مع التكاملات يشتغل فعلاً.

### مبدأ 1 — الشفافية الكاملة
لكل أداة نوضّح للعميل: هل تحتاج توكن/ربط حساب؟ هل فيها اشتراك/تكلفة وكم ومن يدفع؟ + الخطوات خطوة بخطوة («هنا سجّل، هنا انسخ التوكن، هنا بتدفع كذا»).

### مبدأ 2 — خياران للعميل (مرنة)
1. **Self-service (بنفسك بإرشاد):** معالج onboarding يمشي معه خطوة بخطوة ويوضّح وين يحتاج توكن ووين يدفع.
2. **Done-for-you (نسويها لك):** يطلبها منّا كبشر، نؤسس له كل الربط الخاص فيه، برسوم تهيئة إضافية.

### جدول التكاملات

| الأداة | وش تسوي للعميل | تحتاج توكن/حساب؟ | تكلفة/اشتراك | من يدفع |
|--------|----------------|-------------------|---------------|---------|
| **Notion** | مساحة عمل الوكلاء: مهام، سجلات، مستندات | نعم (ربط حساب) | مجاني للأساسي | العميل |
| **Gmail / Outlook** | قراءة/صياغة/إرسال الإيميل | نعم (ربط بريد) | مجاني | العميل |
| **WhatsApp** | استقبال/إرسال رسائل العملاء | نعم (ربط رقم + WhatsApp API) | قد يوجد رسوم API | العميل |
| **Google Calendar** | المواعيد والتذكيرات | نعم (ربط حساب) | مجاني | العميل |
| **Google Drive** | المستندات والملفات | نعم (ربط حساب) | مجاني/حسب المساحة | العميل |
| Slack | تواصل الفريق الداخلي | نعم (ربط حساب) | مجاني/حسب الخطة | العميل (اختياري) |
| Canva | التصميم والمحتوى البصري | نعم (ربط حساب) | اشتراك Canva | العميل (اختياري) |
| **n8n / Make** | محرك الأتمتة وربط الأنظمة | نعم (يديره الفريق) | اشتراك/استضافة | ضمن الخدمة / إضافي |
| **محرك الذكاء (Gemini/Claude/OpenAI API)** | تشغيل الوكلاء أنفسهم | نعم (API key + فوترة) | تكلفة استخدام (tokens) | رسوم اشتراك إضافية شهرية |
| Vercel | استضافة الموقع/الأدوات | نعم (ربط حساب) | حسب الخطة | العميل/الفريق |
| **المنصات الحكومية** (قوى/مقيم/مدد/GOSI) | الامتثال والتنفيذ الحكومي | دخول آمن (بموافقة) | رسوم حكومية منفصلة | العميل (عبر فريق الامتثال) |

### نموذج الرسوم (كيف نكسب والخدمة ما توقف)
1. **رسوم الباقة الشهرية الأساسية** (1,500 / 2,500 / 3,500) — تشمل تشغيل الوكلاء.
2. **رسوم تهيئة (Setup) لمرة واحدة** — إذا اختار العميل Done-for-you. تُحدّد **حسب عدد التكاملات** (سعر لكل أداة تُربط). ✅ قرار باهر
3. **تأسيس اشتراكات الـ APIs = رسوم سنوية** — نؤسس ونغطّي كل اشتراكات الـ APIs/الأدوات للعميل بدفعة **سنوية** (Foundation)، **+ اشتراك شهري إضافي على الخدمة** يضمن استمرارها بدون توقف. ✅ قرار باهر

> التموضع: «خدمة الأتمتة بالذكاء الاصطناعي» = منتج قائم بذاته يُباع فوق الباقة — نؤسس للعميل كل الترابط الخاص فيه، ونعطيه يستخدمه مقابل رسوم الباقة + الاشتراكات الإضافية.

### القاعدة الذهبية للأمان
- **لا نطلب أبداً** كلمة مرور أو OTP. الربط عبر توكنات/OAuth آمنة يربطها العميل بنفسه أو بموافقته الصريحة. أي دفع/اشتراك = بموافقة صريحة ووضوح كامل للتكلفة.

> 📄 التفاصيل الكاملة في صفحة Notion: **🔌 خدمة تهيئة التكاملات والأتمتة (Integrations & Automation Setup)**.

## 🗺️ خطة التنفيذ — من البروتوتايب إلى الحاضنة

### 🔹 المرحلة 0 — إقفال التعريف والأرقام (الأساس) · الآن
**الهدف:** نثبّت التعريف والأرقام قبل ما نبني.
- [ ] تثبيت **نطاق البروتوتايب (MVP)**: البداية بـ **قوى + مقيم فقط** (تجديد إقامات، نقل كفالة، إصدار تأشيرة).
- [ ] اعتماد **نموذج Concierge/هجين**: الوكيل يراقب وينبّه 100% تلقائياً، والتنفيذ الفعلي بمتابعة الفريق يدوياً في البداية.
- [ ] **تصحيح النموذج المالي** ليكون واقعياً + تثبيت الأسعار الثلاث نهائياً.
- [ ] تحديد **رقم هدف عملاء البروتوتايب** (المقترح: 50) و**مدة التجربة** (المقترح: 90 يوم).

**المُخرَج:** وثيقة نطاق + أسعار مثبتة + نموذج مالي واقعي. **معيار النجاح:** كل رقم في العرض له مصدر ومنطق.

### 🔹 المرحلة 1 — بناء البروتوتايب (جاهز للتجربة)
**الهدف:** أقل نسخة قابلة للبيع والتجربة فعلياً مع عميل حقيقي.
- [ ] **Onboarding بسيط**: العميل يربط Notion/واتساب/إيميل بشكل آمن + نموذج بياناته الأساسية.
- [ ] **لوحة حالة + تنبيهات + سجل تدقيق** لكل منشأة.
- [ ] استقرار تجربة **خالد + المتخصصين** على السيناريوهات الأساسية (قوى/مقيم).
- [ ] **بوابة اشتراك/دفع** (يدوي في البداية: فاتورة + تحويل، ثم بوابة لاحقاً).
- [ ] **نموذج طلب/اشتراك تجريبي** على الموقع مربوط بالـ CRM.
- [ ] **Runbook تشغيلي** للفريق البشري الذي يسدّ الفجوة (Concierge) + SOPs.

**المُخرَج:** بروتوتايب حيّ يقدر عميل يشترك فيه ويستخدمه. **معيار النجاح:** عميل واحد حقيقي يكمل الرحلة كاملة (اشتراك → ربط → تنبيه → تنفيذ).

### 🔹 المرحلة 2 — الإطلاق التجريبي مع ~50 عميل (Pilot)
**الهدف:** نثبت الطلب والقيمة على عيّنة حقيقية.
- [ ] **تجنيد ~50 عميل** من الليدز + الشبكة + حملات الإيميل/واتساب (يفضّل بسعر تجريبي/تأسيسي).
- [ ] **Onboarding + متابعة أسبوعية** لكل عميل.
- [ ] **قياس المؤشرات**: التفعيل، الاستخدام الفعلي، الاحتفاظ (churn)، الرضا (NPS)، وقت تنفيذ الطلب الروتيني.
- [ ] **حلقة ملاحظات منظّمة** لتجميع المشاكل والطلبات.
- [ ] **هدف تشغيلي:** صفر حالات تجميد خدمات لعميل مشترك نشط.

**المُخرَج:** بيانات Traction حقيقية + قائمة مشاكل مصنّفة. **معيار النجاح:** عملاء يدفعون + إثبات أن الخدمة تحل ألمهم.

### 🔹 المرحلة 3 — إصلاح الأخطاء والتحسين (Iterate)
**الهدف:** نحوّل ملاحظات الـ 50 عميل إلى منتج ناضج.
- [ ] **تصنيف المشاكل** (تقني/تجربة استخدام/تشغيل/تسعير) وترتيبها بالأولوية.
- [ ] **دورات إصلاح** قصيرة + إعادة قياس بعد كل دورة.
- [ ] **أتمتة تدريجية** لِما كان يدوياً (Concierge → n8n) لتقليل التكلفة.
- [ ] **توثيق الدروس** وتحديث SOPs وقاعدة المعرفة.

**المُخرَج:** نسخة v2 مستقرة + قصص نجاح عملاء. **معيار النجاح:** تحسّن قابل للقياس في الاحتفاظ والرضا مقارنة ببداية الـ Pilot.

### 🔹 المرحلة 4 — تجهيز ملف الحاضنة/المسرّعة والتقديم (Pitch)
**الهدف:** ملف تقديم قوي مبني على Traction حقيقي.
- [ ] **Pitch Deck** (المشكلة، الحل، المنتج، السوق، نموذج العمل، الفريق، الأرقام، حجم الاستثمار المطلوب).
- [ ] إدماج **أرقام الـ Pilot** (عدد العملاء، الإيراد، الاحتفاظ، شهادات) كإثبات جذب.
- [ ] استكمال **حزمة BP3 Incubator Application Pack**.
- [ ] **قائمة الحاضنات/المسرّعات المستهدفة** + المواعيد + متطلبات كل واحدة (داخل + خارج السعودية).
- [ ] **التقديم** + متابعة (وخطاب دعم/تأييد إن لزم لرخصة ريادة الأعمال).

**المُخرَج:** ملف تقديم مكتمل ومُقدَّم. **معيار النجاح:** قبول في برنامج احتضان/تسريع أو وصول لمرحلة متقدمة في التقييم.
