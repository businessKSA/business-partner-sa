## الفكرة: إيجنت واحد بمعنيين

فريق الخدمات المشتركة له **معنيان متكاملان** — نفس الإيجنت من زاويتين:

| | المعنى الأول — داخلي | المعنى الثاني — تجاري |
|---|---|---|
| **الاسم** | قسم 02 — الخدمات المشتركة | طبقة الربط / المكتب الافتراضي |
| **يخدم** | كل أقسام BPIC نفسها | العميل المشترك (منشأة أو موظف فرد) |
| **الدور** | العمود الفقري: GRO Ops، HR Ops، دعم إداري، موردين، مستندات، Onboarding، SOPs | واجهة أمامية واحدة توصل أوامر العميل لكل الفريق المتخصص + إيجنتات المنصات |
| **الملف** | `prompts/shared-services-internal.md` | `prompts/shared-services-product.md` |
| **الـ workflow** | `n8n/internal-request-router.json` | `n8n/subscriber-frontdoor.json` |
| **نموذج البيع** | لا يُباع — يخدم الشركة | اشتراك شهري (الأرقام تحت مراجعة باهر) |
| **الشخصية** | خالد — مدير الخدمات المشتركة | خالد — موظف الاستقبال في "المكتب الافتراضي" |

**قاعدة التوجيه الداخلية:** أي معلومة/خدمة تخدم أكثر من قسم أو أكثر من عميل → الخدمات المشتركة.

**جملة المفتاح من مسودة باهر:** «كل ايجنتس السنيور والجونيور مع بعض = فرقة الخدمات المشتركة» — أي أن المعنى الثاني هو الطبقة التي تجمع كل ما يُبنى في الجلسات الأخرى وتقدّمه للعميل كمنتج واحد.

## علاقة الملفات بنوشن

- يوجد بالفعل **Shared Services Agent — System Prompt** منشور في نوشن تحت صفحة قسم 02 (نموذج Concierge). ملف `shared-services-internal.md` هنا **متوافق معه ويوسّعه**: يضيف منطق التوجيه للفريق المتخصص (فرح/بدر/ملاك/محمد/أحمد) وإيجنتات المنصات، مع أمثلة الرد الآمن AR/EN بنمط GRO الكامل. عند الاعتماد، تُحدَّث نسخة نوشن من هذا الملف.
- تصنيفات الـ router الداخلي مأخوذة من أعمدة **Shared Services Catalog** الفعلية في نوشن (GRO / HR Ops / Admin / Vendors / Documents / SOP / Onboarding / Quality / Success / Templates).

## ملاحظات تقنية (مهمة قبل التشغيل)

1. **Ollama لا OpenAI:** بناءً على نية باهر المعلنة، عقدة الموديل في الـ workflows هي `Ollama Chat Model`. التصميم لا يعتمد على OpenAI API — تبديل الموديل = تبديل عقدة واحدة (sub-node) دون لمس بقية الـ workflow.
2. **Placeholders:** معرّفات قواعد بيانات نوشن (Tasks) ومعرّفات workflows الفريق المتخصص موضوعة كـ `REPLACE_WITH_...` — تُستبدل عند الربط الفعلي. عقد استدعاء الإيجنتات المتخصصة **معطّلة (disabled)** حتى تُبنى في الجلسة الشقيقة، والـ workflow يعمل ويرد بنفسه إلى حين تفعيلها — أي لا حاجة لإعادة هيكلة لاحقاً.
3. **الأرقام التجارية** (5,000 ريال/شهر · 10,000 عميل · 25,000 تأسيس) هي طموح نهائي **تحت مراجعة باهر** — لا يذكرها الإيجنت للعميل كأسعار مثبتة، والـ prompts مكتوبة على هذا الأساس.
4. **Human Approval:** أي التزام مالي/تعاقدي، أي تنفيذ حكومي، أي بيانات دخول/OTP → تصعيد بشري. نفس قاعدة GRO Agent حرفياً.
